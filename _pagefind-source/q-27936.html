Two-tier index design for an append-only text log: (1) per-block rolling-hash fingerprints to filter candidates for each B-block, and (2) a compact suffix automaton over the last W blocks for existence checks. On batch size B, update fingerprints O(B) and extend automaton amortized O(B). A query p runs in O(|p|) to fingerprint and verify. Optimizations: (a) rebuild every K batches; (b) merge small adjacent blocks into larger super-blocks for cache locality. Amortized per char: O(1); per query: O(|p|). Re-scan: O(n) per batch?
