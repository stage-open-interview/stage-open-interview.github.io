You maintain an array a[1..n] of non-negative integers. Updates arrive in batches of size B; each update is either set a[i] = v or remove i (set to undefined). After each batch, output the number of unordered pairs (i<j) with (a[i] + a[j]) mod D == 0 for a fixed D. Propose a data structure to maintain this count with amortized O(B) per batch, and compare to recomputing from scratch. Provide asymptotics in n, B, D?
