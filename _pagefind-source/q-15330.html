Dynamic 2D point set with batched insertions: maintain points in [0,1e6]^2, receiving B new points per batch with no deletions. After each batch, answer Q offline rectangle-count queries reflecting all points up to that batch. Propose a practical data structure that (i) preprocesses once, (ii) handles batch updates amortized, (iii) answers each query in O(log^2 N) or better, (iv) uses near-linear space, (v) includes two optimizations, and (vi) compares to rebuilding a 2D range tree after every batch. Provide asymptotics in n, B, Q?
