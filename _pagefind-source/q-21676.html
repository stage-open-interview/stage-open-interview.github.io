Design a runtime-extendable data transformer pipeline for a JSON ingestion service. Transformers (e.g., EmailMasker, PasswordHasher, DateFormatter) are registered at runtime via a TransformerRegistry and applied in a configurable order per dataset. The pipeline must handle async transformers, avoid mutating inputs in place, allow enabling/disabling by name at runtime, and report per-record transformation errors without failing the whole record. Provide minimal interfaces, a TypeScript usage example, and show how to add a new transformer without redeploying?
