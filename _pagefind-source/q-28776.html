Implement a data structure SlidingWindowTopK that processes a stream of integers. It keeps the last W elements in a fixed-size window using a deque. It must support: add(x), which appends x and evicts the oldest when over W, and topK(K), which returns the K values with the highest frequencies in the current window (ties broken by smaller value). Provide a concrete, beginner-friendly approach using a hashmap for counts, a deque for order, and a min-heap with lazy deletions to maintain the top-K. Include complexity analysis and edge cases?
