You process an append-only log of events where each event carries a numeric ID in [1..U]. Batches arrive contiguously in size B. Maintain two operations: contains(id) -> has this ID appeared? in O(1) amortized; and rank(id) -> number of distinct IDs <= id seen so far. Design an append-friendly data structure that processes each batch in O(B) time, supports queries in O(log D) where D is the number of distinct IDs seen, and uses near-linear space. Provide two optimizations and compare to rebuilding after every batch?
