On a Unix fleet, logs reside under /var/log/app-logs/**, rotated hourly to .log and .log.gz. Each line is either 'YYYY-MM-DD HH:MM:SS LEVEL: text' or a JSON object like {"ts":"...","lvl":"ERROR","msg":"..."}. Multi-line ERROR blocks begin with a line 'ERROR:' and include indented lines. Write a robust one-liner (no external deps) that, for the last 6 hours, reports per-host the count of unique ERROR events, where a unique event is defined by the normalized message text (case-insensitive) and the start timestamp. Must skip unreadable files and files currently open for writing, and must gracefully handle gzipped archives that may be mid-rotation?
