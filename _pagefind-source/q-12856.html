Maintain the top-k elements by value in a fixed-size sliding window of size W as a real-time stream arrives in batches of size B. Each batch adds B new items and, if needed, evicts oldest items to keep the window at W. After each batch, output the current top-k by value. Propose a practical data structure and amortized analysis that (i) supports batch inserts and evictions in O((B) log W), (ii) yields top-k queries in O(k) or O(log k), (iii) uses O(W) space, (iv) includes two optimizations, and (v) compares to rebuilding the top-k by scanning the window after every batch. Provide asymptotics in W,B,k?
