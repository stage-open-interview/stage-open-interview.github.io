Scenario: A PostgreSQL analytics database stores 500GB of e-commerce data with tables: orders(orderId, customerId, productId, quantity, price, status, createdAt), customers(customerId, region, tier, signupDate), and products(productId, categoryId, price, name). The application has these query patterns: Q1) SELECT * FROM orders WHERE customerId = ? AND createdAt > ? ORDER BY createdAt DESC; Q2) SELECT o.* FROM orders o JOIN customers c ON o.customerId = c.customerId WHERE c.region = ? AND o.createdAt > ? AND o.status = 'completed'; Q3) SELECT productId, COUNT(*), SUM(quantity) FROM orders WHERE createdAt > ? GROUP BY productId; Q4) SELECT customerId, SUM(price) FROM orders WHERE status = 'completed' AND createdAt > ? GROUP BY customerId HAVING SUM(price) > ?. Design index strategies for all four queries, explain trade-offs between indexes, and describe how you would validate that indexes are being used correctly by the query planner?
