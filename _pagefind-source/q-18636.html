You have a permutation a[1..n]. Partition indices into consecutive blocks of size K. For every pair of blocks (p<q) maintain inv(p,q) = number of inversions with i in block p and j in block q. After each batch of B updates (i, v) preserving the permutation, update the data structure and report the total number of inversions by summing inv(p,q) over all p<q. Propose a practical sqrt-decomposition design: (i) preprocessing, (ii) amortized update time per updated index, (iii) batch reporting time, (iv) memory, (v) two optimizations, (vi) comparison to recomputing inversions from scratch. Provide asymptotics in n, K, B?
