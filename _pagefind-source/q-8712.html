Design a Kubernetes Operator and CRD named NamespaceGuard to enforce per-namespace data residency. Each NamespaceGuard defines allowedRegion and denyIfNoRegion. The operator should (1) optionally install a MutatingAdmissionWebhook to inject a region-specific nodeSelector into new pods in that namespace, (2) install a ValidatingWebhook to reject non-compliant pods, (3) create a per-namespace NetworkPolicy to confine egress, and (4) expose status, events, and Prometheus metrics. Describe the data model, reconciliation loop, defaulting, migrations when a region is removed, and observability?
