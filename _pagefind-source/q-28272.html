A text S grows only by appending fixed-size blocks of length B. You maintain a suffix automaton for S incrementally, updating it after each batch. To speed queries, you attach a per-block Bloom filter summarizing its substrings. After each batch, given a set of patterns {p1,...,pm}, determine whether each occurs in S efficiently. Derive amortized update cost per appended character and per pattern query, and discuss how B and Bloom filter false positives affect space and cache behavior. Provide asymptotics in n=|S| and m=|{patterns}|?
