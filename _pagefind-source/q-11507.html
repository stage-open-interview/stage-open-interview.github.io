A social media platform uses PostgreSQL for user data and posts, with a compound index on (user_id, created_at) for the posts table. As the platform grows, query performance degrades for two main patterns: fetching a user's recent posts and finding trending posts globally. How would you redesign the indexing strategy, what specific indexes would you create, and how would you handle index maintenance for high write throughput while avoiding index bloat?
