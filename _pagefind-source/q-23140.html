Design a CGOA wrapper for a C function that streams N binary blocks, decompresses each block via a streaming API, and returns a malloc-allocated JSON with per-block latency and checksum. The C API provides a progress callback: void progress(size_t idx, double latency_ms, void* user); The wrapper should expose a Go function: func StreamDecompress(blocks [][]byte, onProgress func(i int, latencyMs float64) error) (string, error). Provide a minimal C header/implementation, a CGO Go binding, and a Go program that uses a worker pool to feed blocks and collect per-block stats with correct memory ownership and error propagation. How would you design it?
