Design a hybrid data structure to support per-user time-window analytics for streaming events. Each event is (user_id: int, action: string, ts: int, value: int). Implement insert(user_id, action, ts, value) and:\n- rangeSum(user_id, actionPrefix, tStart, tEnd): total value for actions that start with actionPrefix for that user within [tStart, tEnd]\n- topKUsers(tStart, tEnd, K): top K users by total value across all actions in [tStart, tEnd]\nConstraints: up to 1e5 users, 1e5 events; timestamps up to 1e9; aim for O(log N) per insert, O(log N) per range query, and near O(K log N) for topK. Use an AVL for user indexing, a Trie for action prefixes, and a Segment Tree over time buckets per user. Explain updates, handling missing data, and memory considerations?
