In a DAG with N nodes and M edges and non-negative edge weights, edges are added only in batches of size B (no deletions). After each batch, determine whether any source-to-sink path has total weight at least W. Design a practical incremental longest-path data structure: (i) maintain a dynamic topological order, (ii) relax only when dist improves, (iii) expose O(1) query via dist array, (iv) use near-linear space, (v) include two optimizations, (vi) compare to recomputing from scratch per batch. Provide asymptotics in n, m, B, W?
