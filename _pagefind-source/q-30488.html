Design a CGOA wrapper for a C streaming API that uses a callback to emit JSON batches. C API: typedef void (*event_cb)(const char* json_batch, void* ctx); void register_event_source(event_cb cb, void* ctx); void start_streaming(size_t batch_size, size_t max_events); The C lib reuses a preallocated arena and requires Go to supply a finalizer to free the allocated batch. Provide: 1) a header and minimal C impl showing memory management; 2) a Go binding exposing StartStream(batchSize int, onEvent func(string) error) error via CGO, ensuring memory ownership and no leaks; 3) a Go worker-pool main that wires the callback to a channel and propagates errors?
