Design a CGOA wrapper for a C streaming tokenizer library. The C API exposes: typedef struct Token { char* text; int type; } Token; Token* next_token(const char* input, size_t len, int* produced, int* eof); void free_token(Token*); The two token types are IDENT (letters) and NUM (digits). Implement: 1) a header and a minimal C impl that tokenizes input by whitespace and returns malloc'ed Token objects; 2) a Go binding exposing StreamTokens(chunks []string) (<-chan string, <-chan error) that feeds each chunk to next_token, copies Token.text into Go strings, frees tokens via free_token, and streams tokens in order; 3) a Go program that runs two streams in parallel and prints tokens.?
