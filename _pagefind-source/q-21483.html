You have a PostgreSQL table posts(id BIGINT, city TEXT, created_at TIMESTAMPTZ, content TEXT). A common query fetches the 20 most recent posts for a city: SELECT * FROM posts WHERE city = $1 ORDER BY created_at DESC LIMIT 20. Another query counts posts per city for a given day: SELECT count(*) FROM posts WHERE city = $1 AND created_at >= $start AND created_at < $end. Design an index strategy using B-tree and discuss a hash index for city equality. Provide concrete index definitions and explain expected EXPLAIN plans?
