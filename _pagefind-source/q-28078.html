Design a beginner-friendly concurrency exercise: a multi-source index updater for a search service. Dozens of producers emit document-change events, each tagged by shard ID. Each shard’s events must be applied in arrival order, while updates across shards can be processed in parallel. Implement per-shard FIFO buffers feeding a bounded global worker queue that writes to a single in-memory index. When backlog crosses a threshold, degrade by coalescing updates per shard to the latest change within a time window and dropping non-critical fields. Explain backpressure, per-shard ordering, and a concrete test plan for correctness, degradation behavior, and graceful shutdown?
