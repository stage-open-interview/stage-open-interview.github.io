Design a runtime-extendable payment validation pipeline for a fintech platform. The core validates a payment request by applying a registry of Rule objects (FraudScoreRule, KYCCheckRule, CurrencyLimitRule) registered by name at runtime. Rules may be sync or async, must run in a defined order, produce uniform error payloads, and isolate per-transaction state. Allow adding/reordering rules without redeploy. Provide interfaces, a registry, and a TS usage example. Compare Pipeline vs Chain of Responsibility trade-offs?
