Design a data structure that behaves as a double-ended queue for integers. It must support pushFront(x), pushBack(x), popFront(), popBack(), and median() which returns the current median value; for even counts, return the lower median. All operations should be amortized O(log n). Explain how you organize storage, handle duplicates, and describe how to maintain the median efficiently when deletions occur from either end (hint: use two balanced multisets or heaps with lazy deletions and rebalancing)?
