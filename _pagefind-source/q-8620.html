In a real-time analytics pipeline, integers arrive in batches of size B from a stream in [0, M-1]. After each batch you must output an online estimate of the total number of distinct values seen so far, with relative error ≤ ε with probability ≥ 1-δ. Propose a HyperLogLog based solution: (i) how to choose number of registers m to meet ε and δ, (ii) per-element and per-batch update times, (iii) memory usage, (iv) how to merge sketches from parallel workers, (v) how duplicates within a batch are handled, (vi) a comparison to exact counting. Express asymptotics in ε, δ, B?
