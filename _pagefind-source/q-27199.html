Design a CGOA wrapper for a C function that hashes a batch of binary blocks using a hardware accelerator with a software fallback. The C API is int hash_batch(const uint8_t** blocks, size_t n, size_t block_len, char** out_json); out_json is a malloc'ed JSON array of objects {"idx":i, "ok":true|false, "digest":"<hex>"}. Provide: 1) a header and a minimal C implementation that uses a software SHA-256 as fallback; 2) a Go binding exposing HashBatch(blocks [][]byte) (string, error) via CGO; and 3) a Go program that streams blocks through a worker-pool, collects per-block digests, and computes a batch checksum (e.g., XOR of digests), with correct memory ownership and error propagation?
