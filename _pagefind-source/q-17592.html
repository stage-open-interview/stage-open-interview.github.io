Maintain an array a[1..n] of integers with online point updates in batches of size B. After each batch, output the length of the Longest Increasing Subsequence (LIS) of the current array. Propose a batched data structure that (i) updates in O(log n) per index, (ii) yields LIS length in O(1) after a batch, (iii) uses near-linear memory, (iv) includes two optimizations, and (v) compares to recomputing LIS from scratch after every batch. Provide preprocessing, amortized per-batch cost, and memory bounds?
