Design a runtime-configurable command handler registry for a chat platform (Twitter/X or Discord). Commands are modular plugins that can be registered by name at runtime, with metadata, versioning, permissions, and cross-command dependencies. The system must support hot-reload without downtime, versioned overrides, and safe fallback when a command is unavailable. Provide minimal interfaces, a registry, and a TypeScript usage example?
