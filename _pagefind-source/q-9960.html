Dynamic forest re-parenting with batch updates: You have a rooted forest with N nodes. Each node has a parent pointer or is a root. Operations arrive in online batches of size B: reattach a node u to a new parent p (ensure p is not a descendant of u to avoid cycles). After each batch, output the total sum of depths of all nodes across the forest. Propose a data structure that (i) preprocesses in O(N), (ii) handles each batch in O(B log N), (iii) can produce the sum in O(1) after each batch, (iv) uses near-linear memory, and (v) includes two optimizations. Compare to recomputing depths via a full DFS after every batch. Provide asymptotics and rough constants?
