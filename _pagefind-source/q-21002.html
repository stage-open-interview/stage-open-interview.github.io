A Unix host stores service logs under /logs/<service>/, rotated hourly into .log and .log.gz. Lines are either YYYY-MM-DD HH:MM:SS LEVEL: text or JSON {ts, lvl, msg}. Multi-line blocks start with ERROR: and indented lines. For a beginner exercise, write a robust one-liner that prints, for each service, the count of distinct ERROR messages observed in the last 6 hours across all files, case-insensitive, whitespace-normalized. Include unreadable files and gz archives; skip them gracefully?
