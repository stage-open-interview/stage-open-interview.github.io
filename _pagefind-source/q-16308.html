A fitness app records workouts across devices. Redis caches the latest 5 workouts per user with a 24h TTL, MongoDB stores user profiles, Cassandra holds time series workouts (user_id, ts, type, duration, calories) bucketed by day, and DynamoDB stores weekly aggregates (user_id, week_start, total_cal, total_dur). For a new workout submission, design concrete write paths, keys, TTLs, and idempotent handling; explain how to compute total calories burned in the last 7 days and refresh the Redis cache; outline a read path to return the latest workout and a 7 day trend?
