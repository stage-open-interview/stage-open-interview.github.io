Design a CGOA wrapper for a C key-value store. The C API exposes: KVStore* kv_open(const char* path, int* err); void kv_close(KVStore*); const char* kv_get(KVStore*, const char* key, int* err); The kv_get return is heap-allocated and must be freed. Expose Go types and funcs: type KVStoreHandle struct{ ptr *C.KVStore }; OpenStore(path string)(*KVStoreHandle, error); func (h *KVStoreHandle) Close() error; func (h *KVStoreHandle) Get(key string) (string, error). Implement ownership so kv_get's C string is freed after Go conversion, and Close frees the store. Demonstrate two goroutines calling Get concurrently. Include header, minimal C, Go binding, and a small program. Discuss error propagation when kv_open or kv_get fails?
