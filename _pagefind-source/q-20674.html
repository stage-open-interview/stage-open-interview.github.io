Design a CGOA wrapper for a C streaming decryption API that uses a callback to deliver decrypted chunks. The C API starts a stream with start_decrypt(const uint8_t* in, size_t n, void (*cb)(const uint8_t* data, size_t len, void* ctx), void* ctx); for each chunk, the callback provides a pointer and length. The Go surface must deliver chunks without copying, with safe memory ownership. Provide: 1) a header and minimal C impl using malloc for per-chunk stats; 2) a Go binding exposing StreamDecrypt(blocks [][]byte) (<-chan []byte, <-chan error); 3) a Go program using a worker pool feeding input and measuring per-block latency?
