On a Unix host, logs live under /logs/** and rotate hourly into *.log and *.log.gz. Each line is either a timestamped text line 'YYYY-MM-DD HH:MM:SS <LEVEL>: <text>' or a JSON line like {'ts':'...','lvl':'ERROR','msg':'...'}, with multi-line ERROR blocks starting with a line 'ERROR:' and indented lines. Write a robust one-liner (no external deps beyond standard UNIX tools) that, for each host directory under /logs, outputs the most frequent ERROR message observed in the last 6 hours, counting multi-line blocks as a single event, across both formats. If there are ties, print all messages with their counts. Include gz archives and unreadable files skipped?
