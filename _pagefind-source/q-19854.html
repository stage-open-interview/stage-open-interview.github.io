Scenario: a real-time analytics service ingests integers in [1..U] in batches of size B every second, maintaining a sliding window of the last W events. After each batch, report the current top-1 value and its frequency within the window. Propose a data structure and an amortized update analysis that (i) handles B inserts and expiry of up to B oldest items, (ii) achieves update cost O(log M) per operation where M is distinct values in the window, (iii) allows top-1 query in O(1) or O(log M), (iv) uses near O(W) memory. Compare to rebuilding frequencies from scratch after each batch, and discuss worst-case scenarios?
