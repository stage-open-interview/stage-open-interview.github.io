Design a CGOA wrapper for a C streaming API that uses a callback to deliver per-block results. The C API is: typedef struct { double value; uint64_t ts; } Block; typedef void (*BlockCB)(const char* json, void* ctx); int process_stream(const Block* blocks, size_t n, BlockCB cb, void* ctx); The wrapper should expose: 1) a C header and a minimal C impl that converts each Block to a malloc-allocated JSON string and calls cb; 2) a Go binding exposing StreamBlocks(blocks []Block) (<-chan string, error) that yields each JSON result; 3) a Go program that streams large sequences of blocks to a worker pool, supports cancellation via context, and ensures all allocated JSON strings are freed after consumption, with proper error propagation?
