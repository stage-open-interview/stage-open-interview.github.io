On a Unix host, logs live under /logs/** and rotate hourly into .log and .log.gz. Each line is either a timestamped line like 'YYYY-MM-DD HH:MM:SS LEVEL: text' or a JSON line like {"ts":"...","lvl":"ERROR","msg":"..."}. Write a robust one-liner (no external deps beyond standard UNIX tools) that outputs 24 lines, each '<YYYY-MM-DD HH>:<count>' representing an hour bucket for the last 24 hours, where count is the number of distinct ERROR messages observed in that hour. Distinctness is by case-insensitive message content; multi-line ERROR blocks start with a line 'ERROR:' and include subsequent indented lines as part of the same event. Include gz archives and nested directories; skip unreadable files?
