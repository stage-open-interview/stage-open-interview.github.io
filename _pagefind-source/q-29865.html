Maintain a dynamic set P of N points in the plane with integer coordinates. Updates arrive in batches of size B; each update inserts a point (x,y) or deletes an existing one. After each batch, determine whether any pair lies within Euclidean distance <= R. Propose a practical data structure and amortized costs: (i) preprocessing in O(N), (ii) updates in O(B) amortized, (iii) query in O(1), (iv) near-linear space, (v) two optimizations, (vi) a comparison to brute-force recheck after every batch. Provide asymptotics in N, B, R?
