Design a runtime-extendable CLI command registry for a Node.js tool. The core ships with 'init' and 'build'. At runtime, new commands (lint, test:watch) can be registered by name without a restart. Each command implements minimal interface { name: string; description: string; execute(args: string[], ctx: RunContext): Promise<void> }. Ensure isolation, parallelizable independent commands, and robust error handling. Provide interfaces, a Registry, and a TypeScript usage example. Discuss trade-offs between plugin vs inline commands?
