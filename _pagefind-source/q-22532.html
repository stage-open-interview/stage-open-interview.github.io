On a Kubernetes node, container logs live under /var/log/containers and rotate into per-container files named *.log and *.log.gz. Each line is either a timestamped text line 'YYYY-MM-DD HH:MM:SS LEVEL: msg' or a JSON line like {"ts":"...","container":"svc-a","level":"ERROR","msg":"..."}; multi-line stack traces start with 'Traceback' or 'Exception' and are indented. Write a robust one-liner (no external deps beyond standard UNIX tools) that, for the last 4 hours, prints per-container the count of distinct ERROR events. Include gz files and unreadable files should be skipped; treat multi-line blocks as single events. Output: <container>.log:<count> per line?
