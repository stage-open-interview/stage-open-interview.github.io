Maintain a dynamic topological order for a directed graph. Build initial order in O(N+M). For each batch of B updates, apply them; if adding edge u->v breaks the order (pos[u] > pos[v]), run a localized DFS from v in the affected region to see if u is reachable (cycle). If not, reposition v to its correct spot. Deletions cannot create new cycles. Amortized per batch: O(B log N); cycle query O(1); memory near-linear. Optimizations: lazy reorder; trigger full rebuild only when frontier grows?
