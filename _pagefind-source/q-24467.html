In a beginner Azure Fundamentals exercise, design a stack for a public portal with a static SPA, a backend API, and a data store. The SPA is served from Azure Static Web Apps; API on Azure Functions (Consumption) behind Azure Front Door with TLS; data in Cosmos DB (SQL API). Add per-tenant isolation via Cosmos DB partition keys, enable Entra ID sign-in for portal users with OAuth2, and implement lightweight API access control (checking an API key header). Include basic monitoring with Application Insights and Azure Monitor alerts, budgeting, and a lightweight DR plan using Cosmos multi-region replication. What deployment plan and trade-offs would you choose?
