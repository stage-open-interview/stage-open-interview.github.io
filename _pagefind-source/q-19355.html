You maintain a DAG with n nodes. Edges arrive online in batches of size B; each batch adds edges (u -> w) but keeps the graph acyclic. After each batch you must output a valid topological order and support O(1) predecessor checks using that order. Propose a practical data structure and amortized analysis that (i) updates the order efficiently with batch inserts, (ii) uses near-linear space, (iii) confines reordering to affected regions with two optimizations, and (iv) compares to recomputing the topological order from scratch after every batch. Provide asymptotics in n, m, B?
