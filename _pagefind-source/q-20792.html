On a Unix host, logs for microservices are in /var/logs/services/**, rotated hourly into .log and .log.gz. Each line is either: a text line like 'YYYY-MM-DD HH:MM:SS LEVEL latency=123ms: message' or a JSON line like {"ts":"...","latency_ms":123,"msg":"..."}. Multi-line traces start with 'TRACE:' and are indented. Write a robust one-liner (no external deps beyond standard UNIX tools; prefer GNU awk) that, for each service, computes the 95th percentile latency of all events observed in the last 2 hours and prints <service>:<latency_ms> in ms, one per line. Include gz archives and unreadable files skipped?
