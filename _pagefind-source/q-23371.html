Maintain a real-time timestamp histogram: N existing events with timestamps in [0, T). Updates arrive in batches of B; each update is either insert t or delete t (t may repeat; deletions guaranteed to exist). After each batch, determine if any bucket of width W contains at least M events. Propose a data structure that (i) preprocesses in O(N), (ii) handles each batch in O(B log S) with S = ceil(T/W), (iii) answers in O(1) whether such a bucket exists, (iv) uses near-linear space, (v) provides two optimizations, and (vi) compares to re-scanning after every batch. Provide asymptotics in N, B, M, W, T?
