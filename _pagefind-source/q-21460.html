Design a CGOA wrapper for a C function that streams data in via a callback. The C API is: typedef void (*chunk_cb)(const uint8_t* data, size_t len, void* user); int stream_process_chunked(chunk_cb cb, void* user, size_t total_chunks, char** out_json); The function calls cb for each chunk and returns a malloc-allocated JSON array of {"chunk":i,"crc32":<value>,"bytes":<len>} and a final status. Provide a header, minimal C impl with malloc for out_json, a Go binding exposing StreamProcess(chunks [][]byte) (string, error), and a Go program that streams chunks via a worker pool while preserving memory ownership and propagating errors?
